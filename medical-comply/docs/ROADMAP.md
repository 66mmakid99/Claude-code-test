## 우선순위 기준

### 우선순위 레벨 정의

| 레벨 | 설명 | 기준 |
|------|------|------|
| **P0** | 긴급 | 서비스 장애, 보안 이슈, 수익 영향 |
| **P1** | 높음 | 핵심 사용자 경험, 주요 기능 |
| **P2** | 중간 | 사용자 가치 증대, 편의성 |
| **P3** | 낮음 | Nice-to-have, 미래 대비 |

### 의사결정 매트릭스

```
         높음  │  P1 Quick Win  │  P0 Must Do   │
   사용자      │                │               │
   가치        │  P2 Consider   │  P1 Plan      │
         낮음  │                │               │
              ─────────────────────────────────────
                    낮음          높음
                        구현 용이성
```

## 피드백 및 요청

### 기능 요청 프로세스

1. GitHub Issues에 기능 요청 등록
2. 레이블 추가: `enhancement`, `priority:*`
3. 월간 리뷰에서 우선순위 결정
4. 로드맵 반영 및 담당자 배정

### 긴급 이슈 처리

1. `priority:critical` 레이블 추가
2. 슬랙/디스코드 채널 알림
3. 핫픽스 브랜치 생성
4. 빠른 리뷰 → 배포
